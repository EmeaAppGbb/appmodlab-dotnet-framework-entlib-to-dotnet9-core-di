## вҡЎ THE LEGACY STACK (BOSS STATS)

**Know Your Enemy!** рҹ‘ҫ

| рҹ’Җ LEGACY BOSS | рҹ“Ҡ POWER LEVEL | рҹҺҜ WEAKNESS |
|---------------|---------------|-------------|
| **Logging Application Block** | Flat file + database logging | ILogger<T> + Serilog |
| **Exception Handling Block** | Named policies, swallows exceptions | Middleware + ProblemDetails |
| **Data Access Block (DAAB)** | ADO.NET + stored procedures | Entity Framework Core |
| **Validation Block** | Attribute-based validation | FluentValidation |
| **Caching Block** | In-memory, no invalidation | IMemoryCache + Redis |
| **Unity Container** | EntLib's preferred DI | Built-in DI |
| **WCF Services** | SOAP, generated proxies | HttpClientFactory + Polly |

### рҹ”Ҙ Anti-Patterns You'll Encounter

```xml
<!-- 300+ lines of XML configuration in web.config рҹҳұ -->
<enterpriseLibrary.ConfigurationSource>
  <sources>
    <add name="System Configuration Source" 
         type="Microsoft.Practices.EnterpriseLibrary..." />
  </sources>
</enterpriseLibrary.ConfigurationSource>
```

- рҹ“ң **XML Config Hell:** 300+ lines of EntLib configuration
- рҹ§ұ **Static Wrappers:** Untestable LogWriter static classes
- рҹ•іпёҸ **Silent Failures:** Exception policies that swallow errors
- рҹҺӯ **Triple Abstraction:** DAAB wrapping ADO.NET wrapping stored procedures
- рҹ”Ң **WCF Spaghetti:** Generated proxy classes everywhere
